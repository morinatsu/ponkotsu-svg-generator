# ぽんこつSVGジェネレーター 仕様書 (統合版)

## 1. プロジェクト概要

「ぽんこつSVGジェネレーター」は、ウェブブラウザ上で直感的なマウス操作（またはトラックパッド操作）により、簡単な図形やテキストを描画し、SVGファイルとしてエクスポートできるシンプルなベクターグラフィックエディタです。

---

## 2. 機能仕様

### 2.1 キャンバスの初期設定
- アプリケーション起動時に、キャンバスの初期サイズ（幅と高さ）を設定するモーダルが表示されます。
- 初期サイズを決定すると、指定された大きさの描画領域（キャンバス）が生成されます。

### 2.2 描画ツール (Toolbar)
ツールバーより以下の描画ツールを選択できます。
- **長方形 (Rectangle)**: キャンバス上をドラッグして任意のサイズの長方形を描画します。
- **楕円 (Ellipse)**: キャンバス上をドラッグして、その矩形領域に内接する楕円を描画します。
- **線 (Line)**: ドラッグの開始点から終了点までの直線を描画します。
- **テキスト (Text)**: 
  - キャンバス上をクリックするとテキスト入力モーダルが開き、任意の文字列を入力して追加できます。
  - テキスト内の改行（`\n`）は、SVGの `<text>` 要素内で複数の `<tspan>` 要素に分割されて描画され、複数行表示に対応します。

### 2.3 インタラクション操作 (Canvas Interaction)

#### 選択と選択解除
- キャンバス上の図形をクリックすると、その図形が**選択状態**（青い枠線または青い操作ガイドが表示された状態）になります。
- 図形以外のキャンバス領域をクリックすると、選択が解除されます。
- 一度に選択できる図形は1つのみです。

#### 移動 (Drag & Drop)
- 選択された図形をドラッグすることで、キャンバス内で自由に移動させることができます。
- `useDragging` フックの制御により、マウスが一時的にキャンバスの外に出てもドラッグ操作が維持されます。

#### 回転 (Rotation - Proximity UI)
- 図形が選択されているとき、バウンディングボックスの**四隅の外側（一定距離：10px〜30px）**にマウスカーソルを近づけると、カーソルが回転モード用（`move` 等）に変化します。
- この状態でドラッグを開始すると、図形の中心点（Centroid）を軸にして回転させることができます。
- `Shift` キーを押しながら回転ドラッグを行うと、**15度刻み**でスナップ（吸着）します。

#### 拡大縮小 (Resizing)
- 図形が選択されているとき、バウンディングボックスの四隅に配置された小さな円形の**リサイズハンドル（内側の10pxエリア）**をドラッグすることで、図形のサイズを変更できます。
- `Shift` キーを押しながらドラッグを行うと、描画開始時の**アスペクト比（縦横比）を維持**したまま拡大縮小されます。
- ※テキスト（Text）図形は拡大縮小をサポートしていません。
- ※直線（Line）図形については、端点（始点・終点）のドラッグ移動による長さ・方向の変更として機能します。

#### テキストの編集
- テキスト図形を**ダブルクリック**すると、テキスト入力モーダルが開き、内容を再編集できます。

#### コンテキストメニュー (右クリックメニュー)
- 図形を右クリックすると、以下の操作メニューが表示されます。
  - **最前面へ移動 (Move to Front)**: 選択図形の重なり順（Z-Index）を一番上にします。
  - **最背面へ移動 (Move to Back)**: 選択図形の重なり順を一番下にします。
  - **削除 (Delete)**: 選択図形を削除します。
- コンテキストメニュー以外の場所をクリックするとメニューは閉じます。

#### 削除
- 選択された図形は、キーボードの `Delete` または `Backspace` キー、もしくはコンテキストメニューの「削除」から削除できます。

## 3. システム仕様とアーキテクチャ

### 3.1 状態管理 (Redux-like State Pattern)
アプリケーションの状態（`AppState`）は、イミュータブルな単一のステートツリーとして管理されています。
状態の更新はアクションのディスパッチ（`dispatch(Action)`）を介して行われます。

- **`mode` (操作モード)**: `'idle' | 'drawing' | 'dragging' | 'rotating' | 'resizing'` のいずれかの状態を取り、マウスのインタラクション制御を切り替えます。
- **サブReducerへの分割**: `src/state/reducers/` 以下に機能単位で Reducer が分割されています。
  - `drawingReducer.ts`: 描画中のプレビューおよび図形確定処理。
  - `draggingReducer.ts`: ドラッグ移動時の座標計算。
  - `rotatingReducer.ts`: 回転ドラッグ時の角度計算。
  - `resizingReducer.ts`: 各種ハンドル位置に基づいたローカル/グローバル座標変換を伴うリサイズ計算。
  - `shapeReducer.ts`: 図形の追加、削除、選択、コンテキストメニュー操作、Z順序変更。

### 3.2 履歴管理 (Undo / Redo)
- `historyReducer.ts`（Higher-Order Reducer）が Undo / Redo の履歴管理を提供します。
- **履歴対象の厳選**: UIの選択状態やツール切り替えといった一時的な状態は履歴から除外され、**キャンバス上の図形データ配列 (`shapes`) に永続的な変更を加えるアクションのみ**が履歴 (`past` / `future`) に記録されます。
- 履歴に記録される代表的なアクション:
  - `END_DRAWING`
  - `DELETE_SELECTED_SHAPE`
  - `CLEAR_CANVAS`
  - `FINISH_TEXT_EDIT`
  - `STOP_DRAGGING`
  - `STOP_ROTATING`
  - (注意: 現状、リサイズ操作完了時の `STOP_RESIZING` は履歴記録ロジックへの統合が Issue となっています)

### 3.3 デバッグモード (Debug Mode)
コマンドラインから明示的にデバッグモードを有効にして起動することができます。
- **起動コマンド**:
  ```bash
  pnpm run dev:debug
  ```
- **仕組み**:
  - `vite --mode debug` によって起動され、環境変数 `VITE_DEBUG_MODE=true` が読み込まれます。
  - 有効時は、アクション履歴を記録する `logger` が適用され、キャンバス下部に現在の State Tree や Undo/Redo 履歴のログを表示する `DebugInfo` コンポーネントが描画されます。

## 付録: バージョン別変更履歴 (Appendix)

- **v0.4.2 (デバッグモードコマンドライン対応)**
  - 環境変数ファイル `.env.debug` を導入し、`pnpm run dev:debug` でアクションロガーと状態ビューア（DebugInfo）を有効化できるように拡張。
- **v0.5 (複数図形対応)**
  - `ShapeData` を Discriminated Union に拡張し、長方形のほか「楕円 (ellipse)」と「線 (line)」の描画に対応。
- **v0.5.1 (Undo/Redo機構の修正)**
  - `historyReducer` を修正し、UIの状態変化を履歴から排除。`shapes` 配列の実質的変更のみを履歴（past/future）に記録するよう最適化。
- **v0.6 (テキストツール追加)**
  - 複数行入力可能なテキストツールとモーダルUIを実装。ダブルクリックによる再編集機能、改行文字列の `<tspan>` レンダリングを導入。
- **v0.7 / v0.7改訂版 (図形の移動機能)**
  - 選択した図形をドラッグ＆ドロップで移動する機能を実装。マウスがキャンバス外に出ても追従を維持するよう `useDragging` カスタムフックを設計。
- **v0.8 (図形の回転機能)**
  - 選択図形の中心を軸とする回転機能を実装。四隅の近傍（10px〜30px）のみで反応する Proximity UI を採用。Shiftキーによる15度刻みスナップをサポート。
- **v0.9 (図形の拡大縮小機能)**
  - バウンディングボックスの四隅にリサイズハンドルを設け、ドラッグで拡大縮小（対角の点をアンカーに固定）できる機能を実装。Shiftキーによるアスペクト比維持をサポート。
- **v0.9+ (コンテキストメニューとZ-Index操作)**
  - 右クリックメニューから図形の重なり順（最前面・最背面）を変更する Z-Index 制御アクションを追加。
