# Dependency Update Report

週次の `dependency-check` ( `pnpm outdated` ) により、以下のライブラリについてバージョンアップが確認されました。
メジャー・マイナー・パッチバージョンアップごとの変更内容の要約を以下にまとめます。

## パッチ・マイナーバージョンのアップデート (自動更新対象)

これらは互換性が保たれているアップデートであるため、今回の対応で更新(`pnpm update`)を実施します。

### 1. `happy-dom` (20.8.3 → 20.8.4) - パッチ
*   **変更の要約**:
    *   `ConsoleConstructor` のインポートをインデックスアクセス型(indexed access type)に置き換えるパッチ修正が行われました。
    *   ([#1845](https://github.com/capricorn86/happy-dom/pull/1845))

### 2. `lint-staged` (16.3.2 → 16.4.0) - マイナー
*   **変更の要約**:
    *   依存関係を減らすために、内部で使用していた `micromatch` をより軽量な `picomatch` に置き換えられました。
    *   ([#1739](https://github.com/lint-staged/lint-staged/pull/1739))

### 3. `typescript-eslint` (8.56.1 → 8.57.1) - マイナー
*   **変更の要約**:
    *   `eslint-plugin`: インターセクション型（交差型）上のプロパティに対して `prefer-optional-chain` ルールが誤って報告を出さないように修正されました。
    *   ([#12126](https://github.com/typescript-eslint/typescript-eslint/pull/12126))

### 4. `vitest` および `@vitest/coverage-v8` (4.0.18 → 4.1.0) - マイナー
*   **変更の要約**:
    *   テストタイムアウトの計測に `performance.now` を使用するようになり、より正確になりました。
    *   モック（Spy）関連の改善：`vi.mocked` での deep partial のサポート追加や、クラスで短縮メソッドを使用した場合の正しいエラーのスローなど。
    *   仮想モジュールに対する `vi.importActual()` の修正。
    *   カバレッジ関連：`thresholds.autoUpdate` 時に末尾の空白を保持する修正や、カバーされていないファイルのトランスフォームモードの推論の修正が行われました。
    *   Browser mode や Chai アサーション等の様々なバグ修正と内部パッケージの依存関係の更新が含まれています。

---

## メジャーバージョンのアップデート (保留・要確認)

以下のパッケージはメジャーバージョンアップ（後方互換性のない破壊的変更を含む可能性があるアップデート）が含まれているため、今回は更新を保留しています。個別に動作検証を行ったうえでアップデートを検討してください。

### 5. `vite` (7.3.1 → 8.0.0) - メジャー
*   **変更の要約**:
    *   内部のバンドラーである Rolldown が v1.0.0-rc.9 にアップデートされました。
    *   `vite-tsconfig-paths` プラグインが検出された際に警告を出すようになりました。
    *   CSS のビルドターゲットとして Lightning CSS を介した `es2025` のサポート追加が行われました。
    *   ブラウザコンソールのログとエラーを開発サーバーのターミナルに転送する機能が追加されました。
    *   その他、すべての非メジャーな依存関係の更新などが行われています。

### 6. `@vitejs/plugin-react` (5.1.4 → 6.0.1) - メジャー
*   **変更の要約**:
    *   **破壊的変更 (Babel 関連機能の削除)**: Vite 8 以降は Oxc によって React Refresh Transform を処理できるようになり、Babel を必要としなくなりました。これにより、プラグインのインストールサイズを削減するため Babel は依存関係から削除され、関連機能が削除されています。
    *   以前のバージョンのように Babel を使用したい場合は、`@rolldown/plugin-babel` を本プラグインと一緒に明示的に導入・設定する必要があります。

### 7. `jsdom` (28.1.0 → 29.0.0) - メジャー
*   **変更の要約**:
    *   **破壊的変更**: 最小サポート Node.js バージョンが v22.12.0+ から **v22.13.0+** に引き上げられました。
    *   CSSOM 実装の刷新：`@acemir/cssom` と `cssstyle` への依存を廃止し、`css-tree` パーサーに基づいた新しい内部実装に置き換わりました。これによりシリアライズ、パース、エッジケースの挙動が改善されています。
    *   `Document` の初期化パフォーマンスが向上し、`Document` ごとに約0.5msのオーバーヘッドが削減されました。
    *   不正なポートのブロック（Fetch 仕様への準拠）、スタイルの変更に伴うメモリリークやクラッシュの修正などが行われています。
